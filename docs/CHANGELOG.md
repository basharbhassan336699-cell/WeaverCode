# سجل التغييرات — WeaverCode 🕸️

## v1.4.2 — حارس الهوية وأدوات التشخيص

### أُضيف
- **`core/engine/query_engine.py`** — **حارس الهوية**: طبقة ثانية تُلحق تذكيراً
  بهوية WeaverCode بنص المستخدم نفسه (وليس حقل system فقط)، فتصمد حتى لو استبدل
  الوسيط/البروكسي حقل system بهوية أخرى مثل «Claude Code». يُتحكَّم بها عبر
  `WEAVER_IDENTITY_GUARD=0/1` (مفعّل افتراضياً).
- **`weaver.py`** — أدوات تشخيص:
  - `--version` / `-v`: عرض إصدار WeaverCode والنموذج والمزوّد وحالة حارس الهوية.
  - `--print-system`: طباعة البروموه النظامي الفعلي المُرسَل للنموذج، للتأكد أن
    هوية WeaverCode تُرسَل فعلاً.
- **`core/ui.py`** — رفع رقم الإصدار الظاهر إلى `v1.4.1+` ليسهل التحقق من نشر
  آخر نسخة على الجهاز (البانر القديم كان يظهر v1.2).

### ملاحظة تشخيصية
- تم التحقق آلياً أن هوية WeaverCode تصل إلى الخادم في حقل system **وفي** رسالة
  المستخدم معاً (صيغتا OpenAI وAnthropic). إذا استمر النموذج في تقديم نفسه كـ
  «Claude Code» بعد تحديث الجهاز فعلياً إلى v1.4.1+، فذلك يعني أن الخادم/الوسيط
  (aerolink) يحقن هوية خاصة به، وهو أمر خارج تحكّم العميل.

## v1.4.1 — إصلاح الهوية في كل الأوضاع

### أُصلح
- **`prompts/system.py`** — إضافة `IDENTITY_CORE`: قاعدة هوية عليا (عربي +
  إنجليزي) تُضاف تلقائياً في مقدمة **كل** الأوضاع عبر `get_system_prompt`،
  وتأمر النموذج بالتعريف عن نفسه كـ WeaverCode فقط، وتمنع منعاً باتاً ذكر
  Claude/Anthropic/GPT أو كشف النموذج/المزوّد، وتتقدّم على أي هوية مُحقونة.
  (سابقاً كان وضع `main` فقط يفرض الهوية؛ أما coding/project/security/
  autonomous/analysis فلم تكن تفرضها إطلاقاً.)
- **`core/engine/query_engine.py`** — `_default_system()` صار يستعمل بروموه
  الوضع الرئيسي (مع قلب الهوية) بدل نص ضعيف لا يمنع انتحال هوية أخرى.
- **`weaver.py`** — الوضع التفاعلي صار يمرّر `get_system_prompt("main")` صراحةً
  بدل الاعتماد على البروموه الافتراضي الضعيف (كان هذا سبب ظهور «Claude Code»
  في المحادثة التفاعلية).

## v1.4.0 — الواجهة الاحترافية (مستوى Claude Code v2)

### أُضيف / تغيّر
- **`core/ui.py`** — إعادة كتابة كاملة لواجهة الطرفية بمستوى احترافي:
  - **شاشة استقبال** بعمودين: معلومات المستخدم (النموذج/المزود/المفتاح/المسار/المحادثات)
    + عمود «آخر النشاطات» (من ذاكرة SQLite) و«ما الجديد».
  - **عنكبوت ASCII** برتقالي بعينين `(oo)` وشبكة، بهوية WeaverCode.
  - **رأس شاشة منقسمة** (`draw_split_header`) أثناء العمل.
  - **مؤشر تفكير متحرك** 🕸️/🕷️ عبر صنف `Spinner` غير متزامن يعمل أثناء
    انتظار رد المزود، ويُفرغ سطره تلقائياً عند طباعة استدعاءات الأدوات.
  - عرض موحّد للأدوات والردود والأخطاء والإحصاءات، وسطر إدخال أنيق `❯`.
  - قراءة الإعدادات من متغيرات البيئة، واحترام `WEAVER_DB_PATH`، وتفادي مسح
    الشاشة عند التمرير عبر أنبوب (غير تفاعلي)، مع الحفاظ على التوافق الرجعي
    لأسماء الدوال القديمة.
- **`weaver.py`** — ربط الواجهة الجديدة: شاشة الاستقبال عند بدء أي وضع،
  رأس الشاشة المنقسمة، ومؤشر التفكير المتحرك في وضعَي المهمة الواحدة والتفاعلي.

## v1.3.0 — دعم مزدوج للمزودين ونقل عبر curl

### أُصلح / تغيّر
- **`core/engine/provider.py`** — إعادة كتابة كاملة لمحرك الاتصال:
  - دعم صيغة **Anthropic** (`POST /v1/messages`) تلقائياً عند اكتشاف
    `aerolink` أو `anthropic` في `WEAVER_BASE_URL`.
  - دعم صيغة **OpenAI** (`POST /chat/completions`) لبقية المزودين
    (OpenRouter، Groq، DeepSeek، OpenAI، Ollama...).
  - استخدام **curl** داخلياً بدل httpx لتجاوز مشاكل إعادة التوجيه (redirect)
    والحالة 305، مع خيارات `follow_redirects` و`--location-trusted`.
  - دعم **بروكسي اختياري** عبر `WEAVER_PROXY` أو `HTTPS_PROXY`/`HTTP_PROXY`.
  - **توحيد الاستجابة**: يُحوَّل رد Anthropic إلى شكل OpenAI (`choices[0].message`)
    فيبقى `query_engine` موحّداً دون تغيير، مع تحويل مخطط الأدوات والرسائل
    (system/tool_use/tool_result) بين الصيغتين.
  - **معالجة كاملة للأخطاء برسائل عربية واضحة** لكل حالات HTTP الشائعة
    (401, 403, 404, 305, 307, 308, 429, 500, 503) وأخطاء الشبكة وJSON.
  - إجبار الصيغة يدوياً عبر `WEAVER_API_FORMAT=anthropic|openai`.
- **`prompts/system.py`** — تشديد قاعدة الهوية: يعرّف الوكيل عن نفسه كـ
  WeaverCode فقط، ولا يذكر أي نموذج أو شركة أو منصة (لا إثباتاً ولا نفياً)،
  والعربية هي اللغة الافتراضية.
- **`config/.env.example`** — شرح الفرق بين صيغتي OpenAI وAnthropic، وإضافة
  **aerolink** كمثال جاهز لمزوّد متوافق مع Anthropic
  (`WEAVER_BASE_URL=https://capi.aerolink.lat`، `WEAVER_MODEL=claude-fable-5`)،
  وإضافة متغيرات `WEAVER_API_FORMAT` و`WEAVER_PROXY` و`WEAVER_FOLLOW_REDIRECTS`
  و`WEAVER_TIMEOUT`.
- **`weaver.py`** — تحميل `config/.env` تلقائياً عند بدء التشغيل، وتحسين قارئ
  `.env` (دعم `export` وإزالة علامات الاقتباس). البانر يعرض اسم WeaverCode فقط.

### ملاحظة
- الآن `python3 weaver.py "قل مرحباً"` يعمل مع أي مزوّد
  (aerolink / groq / openrouter / anthropic / openai) بمجرد ضبط المفتاح في `.env`.

# نظام الذاكرة والسياق في WeaverCode

## كيف يحافظ WeaverCode على السياق ويتجنب النسيان؟

---

## 1. CLAUDE.md — الذاكرة الدائمة بين الجلسات
ملف يُحمَّل تلقائياً في كل جلسة جديدة. يحتوي على:
- تعليمات المشروع الدائمة
- قواعد وأنماط الكود
- سياق المشروع الثابت
- يدعم `@import` لاستيراد ملفات أخرى

**المسارات:**
```
~/.claude/CLAUDE.md          ← ذاكرة عامة (كل المشاريع)
./CLAUDE.md                  ← ذاكرة المشروع الحالي
.claude/rules/*.md           ← قواعد إضافية
```

---

## 2. Auto-Compact — حل مشكلة السياقات الطويلة
عند اقتراب نافذة السياق من الامتلاء، يقوم Claude تلقائياً بـ:
1. تلخيص المحادثة الطويلة
2. الاحتفاظ بالمعلومات الجوهرية
3. مواصلة العمل دون انقطاع

**Hook Events المتعلقة:**
- `PreCompact` — يُنفَّذ قبل التلخيص (تستطيع حقن معلومات يجب حفظها)
- `PostCompact` — يُنفَّذ بعد التلخيص

**أوامر يدوية:**
- `/compact` — تلخيص يدوي للمحادثة
- `/clear` — مسح السياق والبدء من جديد

---

## 3. /resume — استئناف الجلسات السابقة
```bash
claude --resume              # picker تفاعلي للجلسات
claude --resume <session-id> # استئناف جلسة محددة
claude --resume <name>       # استئناف بالاسم
/resume                      # من داخل الجلسة
/rename                      # تسمية الجلسة للرجوع إليها لاحقاً
```

---

## 4. SessionStart Hook — تحميل السياق تلقائياً
يُنفَّذ عند بدء كل جلسة جديدة. يستخدم لـ:
- تحميل نوع المشروع (Node.js, Python, Rust...)
- ضبط متغيرات البيئة
- حقن سياق إضافي في المحادثة

**ملف:** `context-loading/load-context.sh`

---

## 5. Auto-Memory — الذاكرة التلقائية
Claude يحفظ تلقائياً المعلومات المفيدة من المحادثات.
- يُدار عبر: `/memory`
- يُخزَّن في: `MEMORY.md` index
- يُذكَّر Claude بضغطه عند اقتراب الحد

---

## 6. Ralph Wiggum Loop — نظام الاستمرارية
يضمن استمرار Claude في مهام طويلة عبر جلسات متعددة دون ضياع السياق.

**ملفات:** `ralph-wiggum-loop/`
- `ralph-loop.md` — الأمر الرئيسي لبدء الحلقة
- `setup-ralph-loop.sh` — إعداد الحلقة
- `stop-hook.sh` — hook الإيقاف

---

## 7. Conversation Analyzer — تحليل المحادثة
Agent يحلل المحادثة ويستخرج الأنماط والسلوكيات لإنشاء hooks ديناميكية.

**ملف:** `conversation-analyzer/conversation-analyzer.md`

---

## 8. Session State — حالة الجلسة المحفوظة
نظام لحفظ حالة الجلسة في ملف JSON مع:
- File locking (fcntl) لمنع التعارضات
- تنظيف تلقائي للملفات القديمة (30 يوم)
- دعم Remote Sessions

**ملف:** `session-state/session_state.py`

---

## Hook Events الكاملة للذاكرة والسياق

| Event | التوقيت | الاستخدام |
|-------|---------|-----------|
| `SessionStart` | بداية كل جلسة | تحميل السياق |
| `SessionEnd` | نهاية الجلسة | حفظ الحالة، cleanup |
| `PreCompact` | قبل تلخيص السياق | حقن معلومات مهمة |
| `PostCompact` | بعد التلخيص | تحديث الذاكرة |
| `UserPromptSubmit` | عند كل رسالة | إضافة سياق ديناميكي |
| `Stop` | عند إنهاء المهمة | التحقق من الاكتمال |
